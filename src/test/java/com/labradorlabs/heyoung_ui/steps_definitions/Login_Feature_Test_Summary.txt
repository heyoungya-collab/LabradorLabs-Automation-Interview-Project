
* 우선순위 판단 *
로그인 기능은 사용자 접근의 기본이므로 가장 중요하다고 판단하여 먼저 테스트 케이스를 작성하고 실행했습니다.
이 기능이 정상 동작하지 않으면 전체 서비스 이용이 불가능하기 때문입니다. 따라서 사용자 인증이 정확히 이루어지는지, 입력값 검증이 적절한지 집중했습니다.

* 작업 결과 *
테스트 케이스: 정상 로그인, 비밀번호 오류, 존재하지 않는 사용자 로그인, 입력값 미입력 등 총 5개 케이스 작성

발견 이슈: 비밀번호 입력 오타 시 오류 메시지가 불명확함 (재현 절차: 비밀번호 일부 오타 후 로그인 시도)

실행 방법: 테스트 자동화 스크립트는 Selenium과 JUnit 기반이며, IDE에서 mvn test 명령어로 실행 가능

* 다음 단계 *
다음 테스트는 프로그램의 각 모듈에 대한 접근 권한이 정상적으로 부여되어 있는지 확인하기 위한 테스트를 진행할 예정입니다.
이를 통해 사용자가 권한이 없는 모듈에 접근할 수 없는지, 필요한 권한이 있을 경우 원활하게 접근이 가능한지를 검증하려 합니다.
이 테스트는 보안 및 권한 관리 측면에서 중요한 부분이므로 우선순위를 높게 두고 꼼꼼하게 진행할 계획입니다
